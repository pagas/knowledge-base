
**Big O measures how an algorithm's runtime or memory usage grows as the input size (`n`) grows.**

| Big O          | Meaning                                   | Example            |
| -------------- | ----------------------------------------- | ------------------ |
| **O(1)**       | Doesn't grow                              | Access `arr[0]`    |
| **O(log n)**   | Grows very slowly                         | Binary Search      |
| **O(n)**       | Grows linearly                            | Loop through array |
| **O(n log n)** | Efficient sorting                         | Merge Sort         |
| **O(n²)**      | Every item compared with every other item | Nested loops       |
| **O(2ⁿ)**      | Explodes rapidly                          | Recursive subsets  |
| **O(n!)**      | Extremely expensive                       | All permutations   |
